# 工具与技术
```
1、资源日历（用于规划和记录项目资源可用性的工具）
  资源类型：人力、设备、材料
  资源可用性：每个资源在特定时间内的可用性状态。
    例如，人力资源会标明某个员工在特定时间内是否用于项目工作，是全职还是兼职还是休假状态
    对于设备资源，会标明是否可用，是否维护，是否预定。
2、RAM （责任分配矩阵）
  明确每个活动具体的责任人。
3、RACI （负责、批准、咨询、知情）
  明确每个活动每个人的职责，是负责、执行、咨询、还是知情。可以明确每个人的职责，避免职责不清导致的工作冲突、重复、遗漏。
  例如 内部外部资源发生冲突，可以使用RACI。
4、资源分解结构（Resource Breakdown Structure RBS）
  对项目中所需资源进行分类和组织的工具。
5、项目组织图
  以图形的方式展示项目团队成员及其报告关系。
6、冲突管理
  回避：问题推给别人、将问题推到准备充分的时候、从实际或潜在的冲突中退出。
  包容：强调一致而非差异、考虑其他方需要，为维持和谐与关系各退一步。
  妥协：一定程度满意、暂时解决问题或解决一部分问题，容易导致“双输”局面。适用于双方意见的平均值。
  强迫：牺牲一方，推行一方；只提供 输/赢 方案。通常解决紧急问题，立即解决。
  合作：通常会实现“双赢”，综合考虑不同观点意见，使用合作态度、开放式对话达成共识和承诺。

7、团队建设
  改善团队成员之间的配合问题。
  圆桌讨论、研讨会、一对一会议。
  例如：状态审查会上的几分钟议程。非工作环境为改善人与人关系的活动。
7、塔克曼阶梯理论
  改善团队成员之间的协作问题、速度、效率问题。
  例如：如果团队两个成员被替换，团队速度下降，业绩下滑，可以引导团队重新回到组建和震荡阶段。可以改善团队成员之间的相互协作能力。

8、认可与奖励
  用于提升员工积极性。比如可以满足员工的需要。
9、培训
  相较于激励团队成员自己获取知识，培训更加有效。
10、虚拟团队
  
```

# 塔克曼阶梯理论
```
形成阶段
  成员之间相互认识，相互独立，不会坦诚布公。
震荡阶段
  相互对立，凝聚力差
规范阶段
  学习相互信任，协同工作
成熟阶段
  互相依靠，坦诚相待，充满信心
解散阶段
```
