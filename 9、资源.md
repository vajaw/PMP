# 工具与技术
```
1、资源日历（用于规划和记录项目资源可用性的工具）
  资源类型：人力、设备、材料
  资源可用性：每个资源在特定时间内的可用性状态。
    例如，人力资源会标明某个员工在特定时间内是否用于项目工作，是全职还是兼职还是休假状态
    对于设备资源，会标明是否可用，是否维护，是否预定。
2、RAM （责任分配矩阵）
  明确每个活动具体的责任人。
3、RACI （负责、批准、咨询、知情）
  明确每个活动每个人的职责，是负责、执行、咨询、还是知情。可以明确每个人的职责，避免职责不清导致的工作冲突、重复、遗漏。
  例如 内部外部资源发生冲突，可以使用RACI。
4、项目组织图
  以图形的方式展示项目团队成员及其报告关系。
```

# 塔克曼阶梯理论
```
形成阶段
  成员之间相互认识，相互独立，不会坦诚布公。
震荡阶段
  相互对立，凝聚力差
规范阶段
  学习相互信任，协同工作
成熟阶段
  互相依靠，坦诚相待，充满信心
解散阶段
```
